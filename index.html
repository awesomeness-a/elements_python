What is a program?

A program is a sequence of instructions that specifies how to perform a computation.

Basic instructions include:

  • input: get data from the keyboard, a file, or some other device;
  • output: display data on the screen or send data to a file or other device;
  • math: perform basic mathematical operations like addition and multiplication;
  • conditional execution: check for certain conditions and execute the appropriate code;
  • repetition: perform some action repeatedly and execute the appropriate code.

Essentially, a program is a sequence of definitions and commands. Definitions are evaluated and commands are executed by Python interpreter in a shell.

The Python interpreter is a program that reads and executes Python code. To start Python interpreter, type in a shell

python3

This command starts a Python interpreter where you can write Python expressions.

Traditionally, the first program you write is called "Hello, World!"
It's simple and all it does is display the words "Hello, World!" In Python, it looks like this:

# my first Python program print('Hello, World!')

This is an example of a print statement which simply displays a value on the screen.

Statements instruct interpreter to do something. They can be typed directly in a shell or stored in a file that is read into the shell and evaluated.

# symbol indicates that any text to the right is a comment and should be ignored by the interpreter. Comments are useful as notes for the reader of the program.

Congratulations, you just wrote you first Python program! How awesome is that?

Variables and Expressions

One of the most powerful features of a programming language is the ability to manipulate variables. A variable is a name that refers to a value. A value is one of the basic things a program works with, like a letter "a" or a number 42.

An assignment statement creates a new variable and gives it a value.

band_name = 'Black Sabbath' num = 42 pi = 3.141592653589793

To retrieve a value associated with a variable, you simply type its name.

pi >>> 3.141592653589793

The first example assigns a string to a new variable named band_name; the second gives the integer 42 to num; the third assigns the approximate value of Ï€ to pi.

Some of the rules to follow when naming your variables:

  • Try to give meaningful names to your variables, so that when you're back to your own code, you can understand what it is about.
  • Variables can contain letters and numbers, but they can't begin with a number.
  • In Python, we use only lowercase for variables names.
  • The underscore character _ can appear in a name for better code readability.

Also, Python has keywords that can't be used as variable names. If you try to use them as variable names, you will get a syntax error.

>>> class = 'Advanced Theoretical Zymurgy' SyntaxError: invalid syntax

Expressions and Statements

An expression is a combination of values, variables, and operators.

A value all by itself is considered an expression, and so is a variable. These are all valid expressions:

42 n n + 42

A statement is a unit of code that the Python interpreter can execute. We have already seen two kinds of statements: print and assignment.

The important difference between a statement and an expression is that an expression has a value and a statement does not.

Order of Operations

When more than one operator appears in an expression, the order of evaluation depends on the rules of precedence. For mathematical operators, Python follows mathematical convention. The acronym PEMDAS is a useful way to remember the rules:

  • Parentheses have the highest precedence and can be used to force an expression to evaluate in the order you want.
  • Exponentiation has the next highest precedence.
  • Multiplication and Division have the same precedence, which is higher than Addition and Subtraction, which also have the same precedence.

Operators with the same precedence are evaluated from left to right (except exponentiation).

To make the expressions more readable, it's better to use parentheses:
2 + (3 * 4) is easier to read than 2 + 3 * 4.

Strings

A string is an ordered sequence of case-sensitive characters. Strings are basically just a bunch of words.

You can specify strings using single quotes

# A single-quoted string assigned to a variable named pb pb = 'Peanut butter is the glue that holds my life together.'

or double quotes

# A double-quoted string assigned to a variable hold_on hold_on = "Hold on, let me overthink this."

To specify a multi-line string, use triple quotes (either single or double)

# A multi-line string assigned to a variable three_lines three_lines = '''This is the first line of a multi-line string. This is the second line. And finally, the third. '''

Triple-quoted strings are usually used for docstrings. Docstrings are taken as comments when they appear at specific points in your file.

Strings are immutable objects. An immutable object is an object whose state cannot be modified after it is created. It means you can't change its value in place; instead, you must run code to make a new object containing the new value.

Every object in Python is classified as either immutable or not. In terms of the core types, numbers, strings, and tuples are immutable; lists, dictionaries, and sets are notâ€”they can be changed in place freely, as can most objects you'll code with classes.

Sometimes we want to construct strings from other information. This is where the format() method is useful.

year = 1936 name = 'Alan Turing' print('In {0}, {1} published one of the most influential math papers in history, in which he introduced the idea of a "Universal Machine."'.format(year, name))

Output:

In 1936, Alan Turing published one of the most influential math papers in history, in which he introduced the idea of a "Universal Machine."

{0} corresponds to the variable year which is the first argument to the format method, and {1} corresponds to the variable name which is the second argument.

Note that Python starts counting from 0 which means that first position is at index 0, second position is at index 1, and so on.

We can achieve the same result using string concatenation.

'In ' + str(year) + ', ' + name + ' published one of the most influential math papers in history, in which he introduced the idea of a "Universal Machine."'

We can also name the parameters to the format method instead of indexing them:

print('In {year}, {name} published one of the most influential math papers in history, in which he introduced the idea of a "Universal Machine."'.format(year, name))

There is a shorter way to do named parameters, called "f-strings", which is a more elegant and readable way to format a string:

print(f'In {year}, {name} published one of the most influential math papers in history, in which he introduced the idea of a "Universal Machine."')

Escape characters

Let's say you have a quote within your string. If you try to type ' character within your single-quoted string, Python will get confused and give you SyntaxError.

To prevent this from happening, you need to use an escape character. Escape characters invoke an alternative interpretation on the following characters in a character sequence. In this particular case, we use a backslash character:

print('How\'s life?')

Another way to avoid SyntaxError is by using double quotes instead of single quotes.

print("How's life?")

There are a few more escape characters you should know, so go ahead, google about them.

We can check string's length with a built-in len function

>>> name = 'Angela' >>> len(name) 6

To fetch string's components, use indexing expressions

>>> name[0] 'A'

In addition to simple positional indexing, sequences also support slicing, which is a nice way to extract a slice in a single step.

>>> city = 'London' >>> city[2:4] 'nd'

The general form of slicing, x[i : j] means "give me everything in x starting from i and up to but not including j."

Some common usage variations:

>>> city[1:] # Everything past the first 'ondon' >>> city[:3] # Same as city[0:3] 'Lon' >>> city[:-1] # Everything but the last, same as [0:-1] 'Londo' >>> city[:] All of city as a top-level copy, same as city[0:len(city)] 'London'

Lists

Lists are ordered collections of arbitrary objects. They can contain any sort of objects and can be changed in placeâ€”they are mutable objects, and can respond to all the sequence operations used with strings, such as indexing, slicing, and concatenation.

We create a list using square brackets.

new_list = [1, 'apple', 42, 'answer', ['nested', 'list', 'of', 'strings']] empty_list = []

We access list's items by position using the bracket operator. Remember that the indices start at 0

>>> new_list[0] 1

Because they are sequences, lists support many of the same operations as strings. But, unlike strings, lists are mutable.

# Concatenation >>> num_1 = [1, 2, 3] >>> num_2 = [4, 5, 6] >>> num_joined = num_1 + num_2 >>> num_joined [1, 2, 3, 4, 5, 6] # Repetition >>> [0] * 4 [0, 0, 0, 0] # Slicing >>> prices = [45, 87, 94, 67, 89, 100] >>> new_prices = prices[2:] >>> new_prices [94, 67, 89, 100]

List Methods

append() adds a new element to the end of a list

>>> fruits = ['apple', 'orange', 'banana'] >>> fruits.append('dragon fruit') >>> fruits ['apple', 'orange', 'banana', 'dragon fruit']

extend() takes a list as an argument and append all of the elements

>>> veggies = ['carrot', 'potato', 'broccoli'] >>> groceries = ['chicken', 'coconut milk', 'curry sauce'] >>> groceries.extend(veggies) ['chicken', 'coconut milk', 'curry sauce', 'carrot', 'potato', 'broccoli']

sort() arranges the elements of the list from low to high

>>> a = [100, 67, -2, 9, 0] >>> a.sort() >>> print(a) [-2, 0, 9, 67, 100]

List comprehensions are a special way to build a new list by applying an expression to each item in a sequence.

>>> num = [2, 4, 6, 8] >>> squares = [n ** 2 for n in num] >>> squares [4, 16, 36, 64]

Dictionaries

Dictionaries are unordered collections of key:value pairs.

  • They are of the category "mutable mapping."
  • They are objects that map keys to values.
  • Dictionaries can contain objects of any type, and they support nesting to any depth.
  • Keys can be any immutable objects.
  • Sequence operations don't work with dictionaries, because dictionaries are mappings and have no notion of ordering among their items.

We can create a dictionary using several ways.

# Using curly brackets >>> fruits = {'apples': 3.40, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5} >>> fruits {'apples': 3.40, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5} # Using dict() >>> fruits = dict(apples = 3.40, bananas = 2.45, oranges = 4.0, pineapples = 10.5) >>> fruits {'apples': 3.40, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5} # Using lists and zip() >>> fruits = ['apples', 'bananas', 'oranges', 'pineapples'] >>> prices = [3.40, 2.45, 4.0, 10.5] >>> groceries = dict(zip(fruits, prices)) >>> groceries {'apples': 3.40, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5}

In dictionaries, items are stored and fetched by key, instead of positional offset. To fetch a value, you supply the key with which it is associated, not its relative position.

>>> groceries = {'apples': 3.40, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5} >>> groceries['apples'] 3.40

in allows you to test for key existence, and the .keys() method returns all the keys in the dictionary.

>>> 'peaches' in groceries False >>> list(groceries.keys()) ['oranges', 'apples', 'bananas', 'pineapples']

Dictionaries are mutable, so you can change, expand, and shrink them in place without making new dictionaries: simply assign a value to a key to change or create an entry.

>>> groceries['peaches'] = 6.30 >>> groceries {'apples': 3.40, 'peaches': 6.30, 'bananas': 2.45, 'oranges': 4.0, 'pineapples': 10.5}

Other common dictionary methods:

  • .items() returns (key, value) pair tuples
  • .values() returns all the dictionary's values.
  • .get() returns None if key doesn't exist in a dictionary
  • .pop() deletes a key from a dictionary and returns the value it had.
  • .update() merges the keys and values of one dictionary into another, overwriting values of the same key if there's a clash
>>> ingredients = {'eggs': 3, 'spam': 2, 'ham': 1} >>> extra = {'toast': 4, 'muffin': 5} >>> ingredients.update(extra) >>> ingredients {'muffin': 5, 'eggs': 3, 'toast': 4, 'spam': 2, 'ham': 1}

Dictionaries can also be created with dictionary comprehensions.

>>> d = {k: v for (k, v) in zip(['a', 'b', 'c'], [1, 2, 3])} >>> d {'b': 2, 'c': 3, 'a': 1}

Tuples

Tuples work exactly like lists, except that they can't be changed in place (they are immutable) and are usually written as a series of items in paretheses, not square brackets.

  • They are ordered collections of arbitrary objects.
  • They are accessed by offset.
  • They are immutable sequences.
  • You cannot change the size of a tuple without making a copy.
  • Tuples store access points to other objects (references), and indexing a tuple is relatively quick.

A tuple is written as a series of objects, separated by commas and normally enclosed in parentheses.

>>> prices = (45, 32, 4, 98, 76) >>> empty_tuple = () >>> single_item_tuple = (42,) # Tuples concatenation >>> (1, 2) + (3, 4) (1, 2, 3, 4)

The immutability of tuples provides some integrityâ€”you can be sure a tuple won't be changed through another reference elsewhere in a program.

Also, tuples can be used in places where lists cannotâ€”for example, as dictionary keys.

Sets

A set is an unordered collection of unique and immutable objects that supports operations corresponding to mathematical set theory.

  • An item appears only once in a set, no matter how many times it is added.
  • Sets are iterable, can grow and shrink on demand.
  • They may contain a variety of object types
  • Because they are unordered and do not map keys to values, sets are neither sequence nor mapping types.
  • Sets can contain only immutable object types. Lists and dictionaries cannot be embedded in sets, but tuples can if you need to store compound values.

We can create a set using the following:

>>> set([41, 24, 43, 42]) {41, 24, 43, 42} >>> {41, 24, 43, 42} {41, 24, 43, 42} >>> set('apple') {'a', 'e', 'l', 'p'}

Sets support operations corresponding to mathematical set theory.

# Intersection >>> num = {7, 8, 9} >>> num & {7, 10, 9} {7, 9} # Union >>> {1, 5, 3, 6} | num {1, 3, 5, 6, 7, 8, 9} # Difference >>> num - {8, 9} {7} # Superset >>> num > {7, 8} True

Because items are stored only once in a set, sets can be used to filter duplicates out of other collections, though items may be reordered in the process because sets are unordered in general. Simply convert the collection to a set, and then convert it back again.

>>> num = [61, 24, 43, 88, 24, 88, 100] >>> set(num) {24, 43, 61, 88, 100} >>> num = list(set(num)) >>> num [100, 43, 24, 88, 61]

Sets can be used to isolate differences in lists, strings, and other iterable objects.

>>> set([12, 21, 73, 50, 7]) - set([1, 12, 4, 50, 6]) {7, 21, 73}

Sets are convenient when you're dealing with large data setsâ€”the intersection of two sets contains objects common to both categories, and the union contains all items in either set.

>>> bands = {'black sabbath', 'ac/dc', 'thin lizzy', 'jimmy hendrix'} >>> great_guitarists = {'jimmy hendrix', 'eric clapton'}
>>> 'jimmy hendrix' in bands True >>> 'eric clapton' in bands False
# Who is in both? >>> bands & great_guitarists {'jimmy hendrix'}
# All performers in either category >>> bands | great_guitarists {'ac/dc', 'black sabbath', 'eric clapton', 'jimmy hendrix', 'thin lizzy'}
# Performers who are not in the great_guitarists collection >>> bands - great_guitarists {'ac/dc', 'black sabbath', 'thin lizzy'}
# Great guitarists who are not in bands >>> great_guitarists - bands {'eric clapton'}
# Are all great guitarists in bands? >>> bands > great_guitarists False
# Who is in one but not both? >>> bands ^ great_guitarists {'ac/dc', 'black sabbath', 'eric clapton', 'thin lizzy'}
# Intersection >>> (great_guitarists | bands) - (great_guitarists ^ bands) {'jimmy hendrix'}

Files

File objects are considered a core type because they are created by a built-in function, but theyâ€™re not numbers, sequences, or mappings, and they donâ€™t respond to expression operators; they export only methods for common file-processing tasks.

The built-in open() function creates a Python file object, which serves as a link to a file residing on your machine.

# 'w' means open a file in a write mode >>> customers = open(r'C:\customers', 'w')

If the file already exists, opening it in write mode clears out the old data and starts fresh. If the file doesn't exist, a new one is created.

Once you make a file object with open(), you can call its methods to read from or write to the associated external file.

File text takes the form of strings in Python programs; reading a file returns its content in strings, and content is passed to the write methods as strings.

pickle

The pickle module is an advanced tool that allows us to store almost any Python object in a file firectory, with no to- or from-string conversion requirement on our part. It's like a super-general data formatting and parsing utility. To store a dictionary in a file, we pickle it directly:

>>> d = {'apples': 42, 'bananas': 43, 'carrots': 44} >>> f = open('datafile.pkl', 'wb') >>> import pickle >>> pickle.dump(d, f) # Pickle an object to file >>> f.close()

Then, to get the dictionary back later, we simple use pickle again to re-create it:

>>> f = open('datafile.pkl', 'rb') >>> e = pickle.load(f) # Load an object from file >>> e {'apples': 42, 'bananas': 43, 'carrots': 44}

We get back an equivalent dictionary object, with no manual splitting or converting required. The pickle module performs what is known as object serializationâ€”converting objects to and from strings of bytesâ€”but requires very little work on our part.

Pickle internally translates our dictionary to a string form. Because pickle can reconstruct the object from this format, we don't have to deal with it ourselves.

Storing Python Objects in JSON Format

JavaScript Object Notation is a lightweight data-interchange format that is easy for humans to read and write, as well as for machines to parse and generate.

JSON is built on two structures:

  • a collection of name/value pairs
  • an ordered list of values

JSON is a text format that is completely language independent but uses conventions that are familiar to many languages which makes JSON an ideal data-interchange language.

JSON is so close to Python dictionaries and lists in syntax that translation to and from Python objects is trivial, and is automated by the json standard library module.

A Python dictionary with nested structures is very similar to JSON data, though Python's variables and experessions support richer structuring options:

>>> name = dict(first='Bob', last='Smith') >>> person = dict(name=name, job=['dev', 'po'], age=35) >>> person {'job': ['dev', 'po'], 'name': {'last': 'Doe', 'first': 'John'}, 'age': 35}

>>> import json >>> json.dumps(person) '{'job': ['dev', 'po'], 'name': {'last': 'Doe', 'first': 'John'}, 'age': 35}' >>> s = json.dumps(person) >>> s '{'job': ['dev', 'po'], 'name': {'last': 'Doe', 'first': 'John'}, 'age': 35}' >>> p = json.loads(s) >>> p {'job': ['dev', 'po'], 'name': {'last': 'Doe', 'first': 'John'}, 'age': 35} >>> p == person True

It's straightforward to translate Python objects to and from JSON data strings in files. Prior to being stored in a file, your data is simply Python objects; the JSON module recreates them from the JSON textual representation when it loads it from the file:

>>> json.dump(person, fp=open('testjson.txt', 'w'), indent=4 >>> print(open('testjson.txt').read()) {     "job": [         "dev",         "po"],     "name": {         "last": "Doe",         "first": "John" },     "age": 35 } >>> p = json.load(open('testjson.txt')) >>> p {'job': ['dev', 'po'], 'name': {'last': 'Doe', 'first': 'John'}, 'age': 35}

Once you've translated from JSON text, you process the data using normal Python object operations in your script.

Note that strings are all Unicode in JSON to support text drawn from international character sets.

Control Flow
